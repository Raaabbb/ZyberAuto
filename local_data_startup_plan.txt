OVERVIEW
--------------------------------------------------------------------------------

PURPOSE:
Automatically populate Local JSON Storage with mock data when the app starts up, 
allowing immediate testing of features without manual data entry.

TRIGGER:
- App startup (Application.onCreate())
- One-time execution (checked via internal file existence)
- Development/debug builds only (optional: also in release)

TARGET STORAGE:
- Local JSON Files
- Location: json_storage/ directory

2.2 DATA SEEDER SERVICE (Implemented via LocalDataHelper)
    
    The `LocalDataHelper` class handles the data seeding mechanism.
    On initialization, it checks if the `json_storage` directory exists in the app's private storage.
    If not, (or if specific files are missing), it copies the seed files from `assets/json_storage/`.
